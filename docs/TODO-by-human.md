## 공통
- 내 요구사항에 맞게 수정하되, 좀 더 매끄럽고, 격식있고, 자연스러운 표현으로 내 의도를 표현할 수 있다면 너의 생각대로 수정해줘.
- 문장, 단어 등을 추가, 삭제, 변경할 때는 내가 원하는 바를 최대한 반영해줘.
- 문장, 단어 등을 추가, 수정 할때는 디자인에 대한 부분까지도 고려하여 자연스럽게 어울리도록 해줘.

## Hero 섹션
- "연락하기" 버튼 문구가 티도 안나고 와닿지가 않아. 문구와 디자인을 적절하게 수정해줘봐.

## What I Build 섹션
- 공통:
    - 각 Phase 안에 들어가는 항목들중 한개라도 내가 경험한 부분이 있다면 테두리 부분에 자동으로 다른 Phase 와 같이 진행된 부분으로 표현해줘. (예시: Phase 1에서 내가 경험한 항목이 하나라도 있다면 Phase 1 전체가 다른 Phase와 같이 진행된 부분으로 표현)
    - Phase 를 눌러서 나오는 모달에서는 기존 페이지의 스크롤 위치가 유지되도록 해줘. 모달에서 스크롤을 내리면 모달 내부에서만 스크롤이 되고, 모달을 닫으면 기존 페이지의 스크롤 위치가 유지되도록 해줘.
    - 각 Phase 안에 들어가는 항목들이 적절한지, 더 추가되어야할 것은 없는지 검토해서 추가하거나 삭제해줘.
    - 나중에 관리자 페이지에서 각 Phase 안에 들어가는 항목들을 수정할 수 있도록 할 예정이야. 그러니까 그게 가능한 구조인지 검토해서 필요하다면 구조를 수정해줘. 또한 내가 경험한 항목과 미경험한 항목또한 나중에 관리자 페이지에서 수정할 수 있도록 할 예정이야. 그러니까 그게 가능한 구조인지 검토해서 필요하다면 구조를 수정해줘. SW 개발 phase 를 제외하고 나머지 페이지의 모달 안에서, 순차적으로 수행되는 작업과 병령적으로 수행될 수 있는 작업이 랜덤한 배치로 되어있는데, 이런 식으로 배치하는게 괜찮은지 검토해줘. 만약 순차적으로 수행되는 작업과 병렬적으로 수행될 수 있는 작업이 명확하게 구분된다면, 순차적으로 수행되는 작업과 병렬적으로 수행될 수 있는 작업을 구분해서 배치하는게 더 좋을 것 같아. 이런 부분도 검토해서 수정해줘. 이 배치 또한 나중에 관리자 페이지에서 수정할 수 있도록 할 예정이야. 그러니까 그게 가능한 구조인지 검토해서 필요하다면 구조를 수정해줘.

## 기타
- 관리자 페이지를 만드는데 얼마나 큰 노력이 필요한지 검토해줘. 만약 큰 노력이 필요하다면, 일단은 관리자 페이지 없이도 내가 직접 코드 수정해서 각 Phase 안에 들어가는 항목들과 내가 경험한 항목들을 수정할 수 있도록 해줘. 그리고 나중에 시간이 된다면 관리자 페이지를 만들어서 각 Phase 안에 들어가는 항목들과 내가 경험한 항목들을 수정할 수 있도록 해주는 방향으로 가는게 좋을 것 같아. 이런 부분도 검토해서 그게 가능한 구조로 수정해줘.
- 관리자 페이지에서는 자주 수정될 가능성이 높은 부분들을 쉽게 수정할 수 있도록 하는게 좋을 것 같아.
